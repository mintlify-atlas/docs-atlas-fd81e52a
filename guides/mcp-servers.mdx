---
title: "MCP server integration"
description: "Integrate Model Context Protocol servers with your agents"
---

Model Context Protocol (MCP) is an open standard that lets AI agents connect to external tool servers over a common interface. The xpander SDK supports both local (STDIO-based) and remote (HTTP-based) MCP servers through the `MCPServerDetails` model.

## What are MCP servers?

MCP servers expose tools (functions) that agents can call during task execution. Instead of hard-coding integrations, you configure an MCP server once and its tools become automatically available in the agent's `ToolsRepository`.

## Core types

### MCPServerType

```python
from xpander_sdk import MCPServerType
```

| Value | String | Description |
|---|---|---|
| `MCPServerType.Local` | `"local"` | Server process runs locally — SDK launches it as a subprocess |
| `MCPServerType.Remote` | `"remote"` | Server is reachable via HTTP or SSE at a URL |

### MCPServerTransport

```python
from xpander_sdk.modules.tools_repository.models.mcp import MCPServerTransport
```

| Value | String | Description |
|---|---|---|
| `MCPServerTransport.STDIO` | `"stdio"` | Communicate via standard input/output (Local servers) |
| `MCPServerTransport.SSE` | `"sse"` | Server-Sent Events |
| `MCPServerTransport.HTTP_Transport` | `"streamable-http"` | Streamable HTTP (default for Remote servers) |

### MCPServerAuthType

```python
from xpander_sdk import MCPServerAuthType
```

| Value | Description |
|---|---|
| `MCPServerAuthType.APIKey` | Static API key authentication |
| `MCPServerAuthType.OAuth2` | OAuth 2.0 flow |
| `MCPServerAuthType.CustomHeaders` | Custom HTTP headers |
| `MCPServerAuthType._None` | No authentication (default) |

### MCPServerDetails

`MCPServerDetails` describes a single MCP server:

```python
class MCPServerDetails:
    type: Optional[MCPServerType]           # Local or Remote (default: Remote)
    name: Optional[str]                     # Server name
    command: Optional[str]                  # Command to launch local server
    url: Optional[str]                      # URL for remote server
    transport: Optional[MCPServerTransport] # Transport protocol (default: HTTP_Transport)
    auth_type: Optional[MCPServerAuthType]  # Auth method (default: _None)
    api_key: Optional[str]                  # API key for APIKey auth
    client_id: Optional[str]               # OAuth client ID
    client_secret: Optional[str]            # OAuth client secret
    headers: Optional[Dict]                 # Custom headers for CustomHeaders auth
    env_vars: Optional[Dict]               # Env vars for local server process
    allowed_tools: Optional[List[str]]      # Restrict exposed tools (empty = all)
    additional_scopes: Optional[List[str]]  # Extra OAuth scopes
```

## Configuring an MCP server

### Local STDIO server

A local MCP server that the SDK launches as a subprocess:

```python
from xpander_sdk import MCPServerDetails, MCPServerType
from xpander_sdk.modules.tools_repository.models.mcp import MCPServerTransport

mcp_server = MCPServerDetails(
    name="data-server",
    type=MCPServerType.Local,
    command="python",
    transport=MCPServerTransport.STDIO,
    env_vars={"API_KEY": "your-api-key"}
)
```

### Remote HTTP server

A remote MCP server reachable over HTTP:

```python
from xpander_sdk import MCPServerDetails, MCPServerType, MCPServerAuthType

mcp_server = MCPServerDetails(
    name="remote-tools",
    type=MCPServerType.Remote,
    url="https://mcp.example.com/tools",
    auth_type=MCPServerAuthType.APIKey,
    api_key="mcp-server-api-key"
)
```

### Remote server with OAuth 2.0

```python
from xpander_sdk import MCPServerDetails, MCPServerType, MCPServerAuthType

mcp_server = MCPServerDetails(
    name="oauth-server",
    type=MCPServerType.Remote,
    url="https://mcp.example.com",
    auth_type=MCPServerAuthType.OAuth2,
    client_id="your-client-id",
    client_secret="your-client-secret",
    additional_scopes=["read:data", "write:data"]
)
```

<Note>
MCP servers are configured at the platform level (in the xpander.ai console or via the API). Once configured, their tools appear automatically in the agent's `ToolsRepository` — you do not need to register them manually in your code.
</Note>

## Accessing MCP servers on an agent

After loading an agent, inspect its `mcp_servers` property to see which MCP servers are attached:

```python
from xpander_sdk import Agents, Configuration

config = Configuration(
    api_key="your-api-key",
    organization_id="your-org-id"
)
agents_manager = Agents(configuration=config)
agent = await agents_manager.aget(agent_id="your-agent-id")

# Access MCP servers configured for the agent
if agent.mcp_servers:
    for mcp_server in agent.mcp_servers:
        print(f"MCP Server: {mcp_server.name}")
        print(f"Type: {mcp_server.type}")
        print(f"URL: {mcp_server.url}")
```

## Tools from MCP servers in ToolsRepository

Once an agent is loaded, all tools exposed by its MCP servers are merged into the `ToolsRepository` alongside any locally registered tools:

```python
# Access agent's tools repository (includes MCP tools)
tools_repo = agent.tools
print(f"Available tools: {len(tools_repo.functions)}")

# Iterate all tools — MCP and local tools appear together
all_tools = tools_repo.list
for tool in all_tools:
    print(f"Tool: {tool.name} - {tool.description}")

# Retrieve and invoke a specific tool
email_tool = tools_repo.get_tool_by_id(
    "XpanderEmailServiceSendEmailWithHtmlOrTextContent"
)
if email_tool:
    payload = {
        "body_params": {
            "subject": "Test email",
            "body_html": "Hello world",
            "to": ["user@example.com"],
        },
        "path_params": {},
        "query_params": {},
    }
    result = await agent.ainvoke_tool(tool=email_tool, payload=payload)
    print(f"Tool execution success: {result.is_success}")
```

## Tool invocation result

Every tool call returns a `ToolInvocationResult`:

```python
class ToolInvocationResult:
    tool_id: str                 # ID of the invoked tool
    payload: Any                 # Original payload
    result: Any                  # Tool execution result
    is_success: bool             # Success status
    is_error: bool               # Error status
    status_code: Optional[int]   # HTTP status code (if applicable)
    tool_call_id: Optional[str]  # Unique call identifier
    task_id: Optional[str]       # Associated task ID
    is_local: bool               # Whether tool was executed locally
```

<Tip>
MCP servers are automatically loaded when an agent is retrieved. You do not need to restart the SDK or reinitialize the tools repository after adding a new MCP server to the platform — reload the agent to pick up the latest configuration.
</Tip>
